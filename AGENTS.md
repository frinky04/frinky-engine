## Project Snapshot

Frinky Engine is a .NET 8 / C# 12 game engine with a shared core library, an ImGui-based editor, a standalone runtime, and a `dotnet new` template pack.

## Project Layout

- `src/FrinkyEngine.Core/` - shared engine code: entities, components, scene, serialization, rendering, UI.
- `src/FrinkyEngine.Editor/` - desktop editor built with ImGui / Hexa.NET.
- `src/FrinkyEngine.Runtime/` - standalone game runner.
- `templates/FrinkyEngine.Templates/` - template pack.
- `docs/` - hand-written project documentation.
- `AGENT_NOTES.md` - volatile agent-facing pitfalls, workarounds, and implementation gotchas.

## Core Rules

- Keep `AGENTS.md` concise. Put non-obvious workarounds, API pitfalls, and other long-lived agent notes in `AGENT_NOTES.md`.
- If you discover a workaround or future-agent trap that is not obvious from reading the code, update `AGENT_NOTES.md` in the same change.
- Prefer extracting shared logic over adding one-off local fixes.
- Public Core API types and methods need `<summary>` XML docs; Editor and Runtime internals do not.
- Scenes (`.fscene`), prefabs (`.fprefab`), and projects (`.fproject`) are JSON assets.
- Use file-scoped namespaces. Nullable is enabled and `AllowUnsafeBlocks=true`.
- When adding a new system or feature, evaluate whether it should expose console commands or CVars.
- Console registration goes through `ConsoleBackend.RegisterCommand()` / `ConsoleBackend.RegisterCVar(...)`, with engine-wide registration centralized in `src/FrinkyEngine.Core/UI/Console/EngineConsoleRegistrar.cs`.
- Runtime CVar state should live in dedicated static classes such as `RenderRuntimeCvars`.
- CVar naming uses `prefix_name`: `r_` for rendering, `snd_` for audio, `physics_` for physics.
- Register new JSON converters in both `SceneSerializer` and `PrefabSerializer`.
- Never use raw `ImGui.IsKeyPressed()` for editor shortcuts; use `KeybindManager` and `EditorAction`.

## Build & Run

- `dotnet build FrinkyEngine.sln`
- `dotnet run --project src/FrinkyEngine.Editor`
- `dotnet run --project src/FrinkyEngine.Editor -- path/to/Game.fproject`
- `dotnet run --project src/FrinkyEngine.Runtime -- path/to/Game.fproject`
